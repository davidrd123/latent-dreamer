Kernel Data Shapes

Everything is a map. Maps contain maps. Keywords connect them.

world
The single map that holds everything. Every function takes it in and returns a new version.
:contexts the planning tree ▶
;; id -> context map
{:id :cx-5
:parent-id :cx-2 ; -> contexts
:children #{:cx-8 :cx-9}
:all-obs #{{:fact/type ...}} ; visible facts
:add-obs #{...} ; asserted here
:remove-obs #{...} ; retracted here
:ordering 0.85 ; backtrack priority
:rules-run? false
:alternative-past? false}
Contexts form a tree. Each sprout creates a child that inherits the parent's facts. The system explores branches and backtracks when stuck.
:goals active concerns ▶
;; id -> goal map
{:id :g-1
:goal-type :reversal
:planning-type :imaginary
:status :runable
:strength 0.7 ; competition value
:main-motiv :e-shame ; -> emotions
:next-cx :cx-5 ; -> contexts
:backtrack-wall :cx-3}
:main-motiv links to an emotion. That emotion won't decay while this goal is active. :strength determines which goal wins each cycle.
:emotions pressure values ▶
;; id -> emotion map
{:strength 0.7 ; decays 5%/cycle unless motivating
:valence :negative ; or :positive
:affect :shame
:situation-id :s1_seeing_through}
Below 0.15 strength, the emotion is garbage collected. Emotions motivating active goals are protected from decay.
:episodes long-term memory ▶
;; id -> episode map
{:id :ep-7
:rule :rationalization-plan
:goal-id :g-1 ; -> goals
:context-id :cx-5 ; -> contexts
:indices #{:honesty :performance}
:plan-threshold 2
:admission-status :provisional
:rule-path [:rationalization-dispatch]
:anti-residue-flags #{}}
Retrieved by coincidence-mark counting: if enough of an episode's indices overlap with currently active indices, it surfaces. The membrane fields control who can be retrieved and promoted.
:episode-index reverse lookup ▶
;; keyword -> set of episode ids
{:honesty #{:ep-7 :ep-12}
:performance #{:ep-7}
:harbor #{:ep-3}}
This is how retrieval finds episodes quickly. Active indices are looked up here, and episodes that appear enough times cross their threshold.
:needs background drives ▶
;; id -> need map
{:strength 0.6} ; decays 2%/cycle
Slow background pressure. Less important than emotions for cycle-to-cycle behavior but they represent long-term motivational context.

How Maps Reference Each Other

goal → emotion via :main-motiv
goal → context via :next-cx
context → context via :parent-id
episode → context via :context-id
episode → goal via :goal-id
episode-index → episodes keyword → #{ids}
fact → goal via :goal-id in dependency
fact → emotion via :emotion-id in dependency
