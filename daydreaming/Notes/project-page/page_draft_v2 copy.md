# Latent Dreamer

A conducted daydreaming instrument. A human performer steers a character's inner life while a cognitive engine maintains persistent concerns, episodic memory, and involuntary daydreaming between interactions. The stage makes that inner life legible through narration, reactive music, and eventually real-time video.

Earlier rounds built the stage. This round built the mind.

## What Inner Life Looks Like

Four moments from one continuous 12-cycle run. Each beat is generated by an LLM from the kernel's structural state: which concern is active, which operator won, what was retrieved from memory, what emotional pressure is driving the cycle.

<!-- IMAGE: beat 01, cycle 1, rationalization -->
CAPTION: Cycle 1. Rationalization. The kernel selected this operator because negative emotion linked to a failed goal crossed the activation threshold.

<!-- IMAGE: beat 02, cycle 3, reversal -->
CAPTION: Cycle 3. Reversal. The operator changed because the emotional pressure shifted. Reversal imagines a counterfactual past and the prose follows.

<!-- IMAGE: beat 03, cycle 9, roving -->
CAPTION: Cycle 9. Roving. Eight cycles of accumulated residue changed what gets retrieved. The system drifts because it earned the drift.

<!-- IMAGE: beat 04, cycle 12, reversal -->
CAPTION: Cycle 12. The system has been through eleven cycles of rationalization, reversal, and roving. The prose at cycle 12 is structurally different from cycle 1 because the memory is different.

None of this was scripted. The kernel selected operators, retrieved episodes, and stored residue. The LLM inhabited the state it was given.

## Where It Came From

<!-- IMAGE: condensation-v3.jpg -->
CAPTION: From source to substrate. Mueller's 1990 book (left, amber) was condensed into 19 typed mechanism cards (center, white) specifying loop shape, interface boundaries, properties to preserve, and hybrid integration patterns. Those cards drove the build of the hybrid architecture (right, cyan) where Clojure owns loops and state while LLMs supply judgment at bounded sites. Also verified against Mueller's original 1990 Lisp source code.

We found that several of his admission disciplines had been collapsed in early development. Per-index cue roles, hidden episodes, the rule accessibility frontier. We reconstructed them and added the extensions Mueller never needed: a memory membrane for persistent accumulation, anti-residue flags for groove detection, and evidence-driven promotion for durable memory.

## The Memory Membrane

<!-- IMAGE: memory-membrane-v1.jpg -->
CAPTION: How episodes earn their influence. Family plan output enters as provisional (center). Promotion to durable (bottom, amber) requires repeated cross-family success evidence with no contradiction. Demotion (red arrows) fires on stale, backfired, or contradicted episodes. Anti-residue flags (right) detect and suppress self-reinforcing loops. The rule access panel (far right) shows how promotion opens frontier rules to the planner while hard failure quarantines them. Cue zones (bottom bar) separate content indices that drive retrieval from provenance and support metadata that do not. The per-index discipline is reconstructed from Mueller's original Lisp source.

## Rules as Data

<!-- IMAGE: rule-graph-v2.jpg -->
CAPTION: The mind's wiring diagram. Three active family clusters (rationalization in amber, reversal in magenta, roving in cyan) each run a trigger-activation-planning chain. Cross-family bridges (gold lines) connect them: rationalization's afterglow feeds roving, reversal's aftershock feeds both roving and rationalization. These bridges were derived from rule schemas, not hand-coded. The planner-visible zone (bright center) is what the planning engine can see. The frontier (middle ring) is where serendipity searches for rules the planner does not know about yet. Quarantined rules (outer edge) are suppressed by hard failure evidence.

## What This Round Built

This is Round 3 of a hackathon project. Earlier rounds built the live performance surface: Scope for real-time video at 24fps, an APC Mini MK2 repurposed as a cognitive controller, Lyria RealTime for reactive audio, LoRA style morphing, depth conditioning.

This round built the mind that will feed that stage.

A Clojure kernel running Mueller's control loop with five daydreaming families. A rule engine with typed rules, a custom matcher, and a connection graph that derives structural edges from rule schemas. A memory membrane with admission tiers, anti-residue flags, evidence-driven promotion, and a rule accessibility frontier. A typed executor boundary where family plans return declarative effects and the kernel validates and applies them. A runtime thought projector and a feedback loop where generated thought changes future cognition. 19 mechanism cards condensed from Mueller's book, verified against his original Lisp source. Five rounds of external architecture review absorbed into the build. 241 tests. 1242 assertions. All passing.

## What Is Proven

Inner-life prose driven by persistent cognitive state, not just a prompt. The feedback loop changes later cognition, with traces diverging from cycle 4 onward. The memory membrane prevents self-reinforcing grooves. Rules-as-data enables structural bridge discovery across families. The generation pipeline produces material that survives traversal. First Graffito microfixtures with typed psychological facts show same-situation reappraisal working mechanistically.

## What Is Not Proven Yet

Full serendipity traversal (connection graph exists, path verification does not). Cross-session accumulation (the hard falsification test). The first real LLM-backed rule (designed, not built). A full staged run with narration, audio, and visuals together. Rich authored worlds beyond microfixtures.

## What It Enables

A performance instrument with an inner life. A human performer steers pressure, mood, and mode while the character's cognitive machinery keeps working. The stage becomes a legible display of thought.

Prosthetic inner life for language models. Between interactions, the model can rehearse, rationalize, avoid, roam, and remember. You come back and the character is different because it has actually been through something computationally.

Accumulation across sessions. If the architecture works, the system gets more capable by operating. The memory membrane ensures that accumulation is earned, not self-reinforcing.

---

## The Honest Claim

This is a foundation, not a finished instrument. The architecture is designed, partially built, and benchmarked. The dynamics work in miniature. The discipline holds under test.

The bet: persistent typed cognitive structure, owned by an engine and augmented by bounded LLM judgment, will accumulate in ways no prompt-only system can match.

If later sessions do not become structurally different because of what earlier sessions discovered, the architecture does not earn its keep.
