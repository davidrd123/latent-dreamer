# Collaborative Pivot Synthesis

Last updated: 2026-03-24

This note is the compressed synthesis of the recent pivot discussion.
It is not a sprint contract. It is the current best read on what the
project seems to be becoming, what the current kernel genuinely proves,
what it does not yet prove, and what other work in the repo cluster is
teaching us about the next layer.

---

## One-sentence shift

The project is no longer best understood primarily as a conducted
daydreaming instrument or as a Mueller-faithful recapitulation.
It is becoming a substrate for persistent collaborative inner life:
something that can think with a human over time, accumulate state with
discipline, and bring that state back into future work.

The current Mueller-shaped kernel is still valuable.
It now looks more like a substrate and test bed than a final ontology.

The clearest current product split is:

- **product**: issue-centered studio partner
- **internal design**: polyphonic daemon
- **mode**: conducted inner-life instrument

## What the kernel genuinely has

These now look load-bearing beyond the original performance framing.

| Mechanism | Verdict | Notes |
|-----------|---------|-------|
| **Concern competition** | Core — keep and widen | One attention economy is the central good idea. Widen concern sources beyond failed goals + negative emotion. |
| **Memory membrane** | Core — keep hard | Admission tiers, cue zones, anti-residue, promotion by later use. Among the best things in the repo. |
| **Coincidence-mark retrieval** | Core — keep as one retriever | Multi-signal convergence is a real differentiator over embedding blur for surprising-but-grounded returns. Make it one retriever among several, not the only one. |
| **Branchable contexts** | Valuable — likely lighten | Full Mueller branching is overkill for day-to-day collaboration. Shrink into lighter branch workspaces: "what if," "draft alt," "counterexample," "conversation rehearsal." |
| **Typed internal moves** | Valuable — keep as seed library | Some notion of "what kind of thinking is happening" still matters. Keep families as a seed library of moves, not the master schema. |
| **Path dependence / writeback** | Core — keep | What happened changes what can happen next. The system is not purely stateless. |
| **Carried transient state** | Core — transfers directly | The Graffito Tony-state reread result is exactly right. The same material should reread differently depending on current stance, energy, confusion, urgency. |
| **Family taxonomy** | Valuable now, likely superseded | Too Mueller-shaped as a universal ontology. Rehearsal, reversal, repercussions transfer well. Rationalization should survive as an explicit, risky move. Revenge is baggage. |
| **Rule connection graph** | Replaceable implementation | Keep typed graphs, but demote the rule graph from center stage. The human-facing graph should be issues, claims, evidence, preferences, branches. |
| **Performance/daydreaming oscillation** | Likely historical baggage | The real distinction is useful but rename it: foreground collaboration / background incubation / timed resurfacing. |

At a product level, the kernel is already fairly strong at:

- rumination / persistent preoccupation
- counterfactual branching
- memory discipline under accumulation

## What the kernel does not yet have

The recent 5 Pro / chat material converges pretty hard here.

### 1. Epistemic state

The kernel mostly has episodes. A collaborative companion also needs
first-class objects like:

- claims
- questions
- hypotheses
- evidence
- contradictions
- commitments

Without those objects, the system can keep circling concerns without
cleanly saying what is believed, doubted, contested, or unresolved.

### 2. Relational state

The kernel has endogenous state and world state. It barely has state
about the relationship between human and system.

Examples:

- what to keep warm
- what to cool or back off from
- what was promised for later
- what kind of interruption is welcome
- where trust broke
- what the system keeps misreading

This now looks like one of the largest missing layers.

### 3. Richer concern sources

The kernel still leans heavily on failed goals plus negative emotion.
That is too narrow for a collaborator.

Future concern sources likely include:

- contradiction / dissonance
- curiosity
- social debt
- promise / obligation
- aesthetic pull
- boredom / staleness
- "this feels important but I do not know why"

### 4. Human uptake as signal

The membrane is currently strongest on internal use/outcome evidence.
For collaboration, some of the strongest promotion or suppression
signals may come from human uptake:

- followed later
- reopened later
- copied into a draft
- explicitly rejected
- frozen / kept warm
- dismissed / cut away

### 5. Conversation repair

The kernel has planning backtracking. It does not yet have a good model
of collaborative repair:

- wrong level
- wrong timing
- wrong initiative
- wrong interpretation
- wrong tone or pressure

That matters more for companionship than many memory failures do.

### 6. Common ground and return timing

Collaboration needs more than recall. It needs explicit records of:

- what is settled
- what remains open
- what terms mean for current purposes
- what changed our minds
- what is intentionally postponed

It also needs a resurfacing policy:

- when to knock
- whether something deserves interruption
- whether it should wait for a more relevant context

This is the missing bridge between between-session thinking and useful
initiative.

### 7. Transformation lineage

The system needs to remember not only that something was used, but how
it changed:

- vague discomfort -> named contradiction
- fragment -> hypothesis
- hypothesis -> abandoned due to counterexample
- repeated avoidance -> explicit issue

That is a stronger collaborative trace than episodes alone can provide.

### 8. Contestation and taste

The system needs real counterpressure, not just support and retrieval.
It should be able to say: this conflicts with your earlier claim, this
is probably rationalization, here is the strongest alternative reading.

It also needs durable, revisable preference structures: preferred
argument shapes, tolerated ambiguity, favored explanatory styles, what
kinds of disagreement the human finds fruitful. Without taste, the
system can remember facts while remaining stylistically hollow.

### 9. Polyphony

Single-self architecture is the wrong abstraction for a persistent
collaborator. A small ecology of voices with different rights — some
generate, some critique, some interrupt, some only annotate — changes
failure modes. One monolithic companion slides toward flattery. A
chorus can disagree internally before bothering the human.

## What the Graffito miniworld just taught us

The recent emotional-state diagnosis is not a side bug. It is a
category lesson.

The miniworld is a hybrid harness:

- some behavior is real kernel behavior
- some state is benchmark-carried
- some facts are benchmark-projected
- some values are derived appraisal output

That means the trace can visually imply stronger dynamic claims than
are actually warranted. In particular, some Graffito emotions are
currently scene-anchor projections rather than live decaying affect.

So the miniworld still proves real things:

- family execution
- reuse and promotion
- membrane flags
- Tony-state carryover
- same-situation reappraisal

But it does **not** yet prove the full emotional-dynamics story that
the trace can suggest.

This matters for the pivot because it sharpens a standard:

**state should operate, not just decorate.**

If a state distinction only colors narration or labels a scene, it is
not yet first-class enough.

## What Symbiotic-Vault adds

John's vault work is not a replacement for the kernel.
It is a spike on the layers the kernel currently lacks.

### The strongest imports

- **Separate use mode from dev mode**
  This is a clean answer to a confusion we keep hitting: engineering
  the system is not the same thing as collaborating with it.

- **Separate human surfaces from agent surfaces**
  The vault's "agent never writes into human surfaces" rule is a strong
  relational design principle.

- **Propose, don't impose**
  Suggested structure should stay suggestive until the human ratifies
  it.

- **Reflection as relationship-building**
  The `reflect` skill is not extraction or organization. It is the
  agent building accumulated familiarity with the human's thinking.

- **Threads + Candid + reply loop**
  This is one of the best concrete models of joint unfinishedness in
  the repo cluster.

- **Typed epistemic objects**
  The vault's atom kinds — concept, claim, method, question, person,
  reference, event — are a much better collaboration substrate than
  episodes alone.

- **Interactive curation signals**
  Freeze, dismiss, respond, and cut are concrete human-uptake signals.
  They look like the interface-side half of the membrane idea.

### The strongest caution

The vault has much richer relational and epistemic scaffolding, but
almost no true attention economy. Everything is broadly available until
human or agent selects it. That means it complements the kernel rather
than superseding it.

## Clean synthesis

The most useful reframe so far is:

- **Concern state**: what wants processing
- **Epistemic state**: what is believed, questioned, contradicted, evidenced
- **Relational state**: what is live between human and system

The current kernel has a strong start on the first, fragments of the
second, and almost none of the third.

The vault work has much more of the second and third, but little of the
first in algorithmic form.

So the interesting direction is not "replace one with the other."
It is:

**keep the kernel's attention economy and memory discipline, while
expanding the object model and collaboration surface toward epistemic
and relational state.**

## Where LLMs sit now vs. where they need to sit

Mueller's DAYDREAMER spent ~46% of its code on representations and
planning rules — the common-sense knowledge layer. The cognitive
architecture (control, episodic memory, reversal, serendipity,
mutation) was only ~28%. LLMs invert that ratio: effectively unlimited
common-sense knowledge, zero cognitive architecture for persistence,
attention competition, or memory discipline.

The kernel currently uses LLMs at bounded edges only:

- `runtime_thought.clj` — content realization from cycle state, plus
  a small "what felt unresolved?" judgment via residue writeback
- `director.clj` — small scene-level nudges: concepts, boosts,
  emotional episodes, interpretation notes
- `family_evaluator.clj` — post-hoc triage of already-produced family
  plans (realism, desirability, retention)

The dominant pattern is LLM-as-evaluator, not LLM-as-core-generator.
LLMs do not yet build the world model, drive the concern/goal/family
loop, own memory promotion, own retrieval strategy, or execute
LLM-backed rules (that path in `rules.clj` is still unimplemented).

So the common-sense layer that Mueller hand-coded is mostly not there
yet as a first-class architectural layer. The Graffito miniworld fills
that gap with hand-authored benchmark logic.

### The practical bridge

Not all of Mueller's 135 rules need LLM replacements. The key entry
points are:

1. **Concern initiation** — "what feels unresolved?" Mueller had
   ~12 theme rules. This is the smallest surface, highest value.
   One prompt, one typed output format, one validation step.
   Everything downstream (competition, retrieval, membrane) runs
   on existing machinery.

2. **Evaluation / contestation** — "does this conflict with
   anything?" Also small surface, directly serves the epistemic
   collaborator direction.

3. **Planning decomposition** — "how would you approach this?"
   Mueller had 130 planning rule applications in one trace. This is
   the big surface and the hard one. Save for later.

The cheapest first experiment is probably #1: keep the kernel's
control loop and memory exactly as they are, but let an LLM propose
what enters the concern economy. The kernel validates, competes,
retrieves, and membranes. The LLM supplies the content that currently
comes from hand-authored Graffito templates.

This is the concrete meaning of the proposer/committer split applied
to the common-sense gap.

## Product identities not to collapse

Three identities are currently mixed together.

### A. Rumination daemon

What keeps coming back?
What gets avoided?
What reenters under pressure?

The current kernel is already fairly good here.

### B. Epistemic collaborator

What do we think, doubt, contest, owe, and revise together?

This is where the personally useful direction seems to live.
The current kernel is weak here.

### C. Counterfactual studio

What branches remain live?
What abandoned alternatives deserve reopening?

The current kernel is strong here because of contexts and reversal.

The category error to watch is continuing to polish A and C while
telling ourselves we are building B.

## What this means right now

### Do not do

- do not tear the kernel down
- do not declare Mueller irrelevant
- do not confuse a public ontology with an internal control vocabulary
- do not pretend the collaborative layer will emerge automatically from
  more miniworld elaboration

### Do keep doing

- understand the existing code end to end
- improve trace / canopy visibility and provenance labeling
- keep the current mechanistic branch honest
- use the kernel as a substrate and falsifiable test bed

### What to question harder

- whether the family taxonomy should remain internal control/debug
  vocabulary rather than public ontology
- whether episodes should stay the only major persistent object
- how porous the kernel/LLM boundary should become
- whether the right split is:
  - `LLM = strong proposer, weak committer`
  - `Kernel = weak proposer, strong committer`
- how human uptake should feed back into memory authority
- what the smallest real relational-state layer could be

## Working thesis

The project now seems best aimed at:

**a persistent collaborative companion with disciplined inner life**

not just:

- a performance instrument
- a character engine
- a Mueller reconstruction

The current build remains useful because it already solves some of the
hardest substrate problems:

- attention economy
- persistence
- suppression
- branching
- nontrivial memory authority

The next large architectural win is unlikely to be "one more family."
It is more likely to be:

- first-class epistemic objects
- first-class relational state
- common-ground tracking
- return-ticket / resurfacing policy
- human uptake entering the membrane
- better provenance over what is carried, projected, and derived

If the proposer/committer split is right, the boundary loosens here:

- LLM proposes new issues, contradictions, hypotheses from raw sessions
- LLM reformulates retrieval (ask for counterevidence, broader/narrower)
- LLM proposes schema pressure ("your current node types are failing here")
- LLM instantiates competing partial readings (polyphonic debate)
- LLM writes return tickets, marginal notes, rehearsal snippets

And stays strict here:

- kernel owns durable memory promotion
- kernel owns shared-ground updates (LLM can propose "this seems settled")
- external actions stay gated
- LLM is not sole judge of whether its intervention was valuable

One useful design image for that future system is:

**a synthetic fringe**

A live edge of half-formed returns around the work:

- contradictions that finally crossed threshold
- stronger names for old discomforts
- counterexamples to favorite stories
- rehearsal lines for avoided conversations
- analogies that only become visible after weak cues converge

## Adjacent traditions worth importing

- **Discourse graphs / Joel Chan** — questions, claims, evidence, sources
  as base objects; ordinary outlining generates typed edges
- **Synergi** — mixed-initiative scholarly synthesis; editable threads,
  not answer generation
- **Graphologue** — LLM responses become editable node-link diagrams;
  non-linear human-LLM dialogue surface
- **Clark & Brennan / grounding** — communication as cooperative action;
  what "understood well enough for current purposes" means
- **Argumentative human-AI decision-making** — agents should reason
  with us, not for us; contestable, revisable dialogue
- **George Lewis / Voyager** — independent internal process in a
  nonhierarchical partnership; the medium is unresolved thought
- **IBIS / design rationale** — issue-centered collaboration
- **Bakhtin** — polyphony
- **Winnicott** — transitional space without reality confusion
- **IFS** — parts with different agendas
- **Tana** — familiar outlining on a graph substrate
