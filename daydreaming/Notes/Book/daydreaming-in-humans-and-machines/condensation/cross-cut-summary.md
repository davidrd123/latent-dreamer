# Cross-Cut Summary

One-page summary over the full 19-mechanism set. This compresses each card to the fields that matter most for architecture triage: current kernel coverage, strongest hybrid cut, and the property that modernization must not erase.

| ID | Mechanism | Kernel status | Strongest hybrid cut | Property to preserve |
| --- | --- | --- | --- | --- |
| 01 | Emotion-driven control | Partial | None inside scheduler; adjust concern pressure upstream | One global arbitration point over persistent concerns |
| 02 | Need decay / emotion decay | Implemented | None | Different state types need different persistence profiles |
| 03 | Concern initiation | Partial | Motivating-emotion assignment when rule output is underspecified | New goals become explicit persistent concerns tied to context |
| 04 | Concern termination | Partial | None | Concern ending must leave explicit aftermath |
| 05 | Planning rule application | Partial | Soft applicability plus retrieved-episode ranking | Planning stays explicit rule/context dispatch, not one-shot generation |
| 06 | Inference rule application | Not recovered | Soft matching plus uncertain-consequent scoring | Inference remains explicit local fixpoint reasoning with provenance |
| 07 | Subgoal creation | Partial infrastructure | Procedural consequents only | Child goals, branches, and bindings stay explicit |
| 08 | Context sprouting / backtracking | Substantially implemented | None | Alternatives evolve in ancestry-linked world states |
| 09 | Analogical rule application | Not recovered | Analogy aptness plus repair-vs-reject evaluation | Analogy remains explicit source-to-target transfer with repair points |
| 10 | Episode storage | Partial | Extra semantic indices and richer summaries | Stored experience stays decomposable and multiply indexed |
| 11 | Episode retrieval | Substantially implemented | None in core loop; auxiliary cue generation only | Retrieval preserves which indices matched |
| 12 | Reminding cascade | Substantially implemented | Salience promotion after reminding | Individual indices propagate through bounded recency structures |
| 13 | Serendipity recognition | Not recovered | Path usefulness plus soft verification | Rule graph stays structurally derived, not usage-weighted |

| 19 | Analogical repair cases | Partial, uneven | Repair-target ordering plus keep/discard judgment | Repair families preserve explicit temporal and causal orientation |
