# Missing Engine Map

Last updated: 2026-03-23

Grounded in:
- `daydreaming/Notes/current-sprint.md`
- `daydreaming/Notes/dashboard.md`

This is the short answer to "what is still missing?" after the memory
membrane, executor seam, Assay A/B, and the first Graffito miniworld.

The main distinction to keep straight is:
- some gaps are **immediate mechanistic next work**
- some are **next substrate abstractions**
- some are the **deep engine** Mueller had or wanted
- some are **product / world proof** gaps

---

## Now

These are the active missing pieces inside the current Graffito /
memory-ecology branch.

- Broaden the Graffito miniworld beyond the current
  rehearsal-targeted rationalization paths.
- Keep the miniworld running longer and inspect whether retrieval,
  provenance, reread, and promotion remain legible over time.
- Extend attributed use / outcome handling beyond the current narrow
  source-loop slices.
- Broaden vindication policy beyond the current source / stale / loop
  paths.
- Widen outcome resolution beyond simple
  `:succeeded` / `:failed` / `:backfired` / `:contradicted`.
- Strengthen anti-residue and consolidation beyond the current
  first-pass membrane.
- Add more graph density:
  evaluation facts, retrieval events, critic signals, richer bridge
  evidence.

This is still the current mechanistic frame. The honest question is no
longer "can the kernel do promotion / frontier opening at all?" It is
"how does that behavior broaden and stay legible as the miniworld gets
less narrow?"

---

## Next

These are the next substrate abstractions after the current
memory-ecology / Graffito pass settles.

- Verified path layer:
  `candidate -> projection-verified -> episode-constructed`
- Generic `:clojure-fn` executor dispatch
- First real `:llm-backed` content-generation rule
- Stronger generic dispatch / inference plumbing after verified paths
- Clearer kernel-owned verifier discipline at those judgment points

The control-plane order still matters:
- strengthen memory ecology and consolidation first
- then build verified paths and generic dispatch

---

## Later

These are the deeper creative-engine mechanisms that are still mostly
designed, traced in Mueller, or partially scaffolded, but not yet
implemented as the live engine.

- Full serendipity search over the heterogeneous graph
- Analogical planning / analogical replanning from stored episodes
- Mutation as a real plan-transform mechanism, not just logged branch
  residue
- Relaxed planning / daydreaming-mode constraint loosening
- Richer reusable planning-tree structure for adaptation
- Regulation-modulated serendipity thresholding
- REPERCUSSIONS as a more developed future family, if and when it
  becomes worth promoting

This is the layer where the engine becomes genuinely generative rather
than mainly disciplined accumulation plus family-specific reuse.

---

## Still Missing At Project Level

These are not just kernel gaps. They are project-proof gaps.

- Cross-session persistence
- A full staged run with conductor + kernel + narration/audio/video
  working together
- Richer authored worlds beyond the first mechanistic Graffito
  miniworld
- External proof that someone outside the architecture loop sees and
  responds to the result

---

## Short Version

If someone asks "what are we still missing?" the compact answer is:

1. Right now:
   broader Graffito dynamics, richer use/outcome/vindication policy,
   stronger consolidation, more graph density.
2. Next:
   verified paths, generic dispatch, the first honest `:llm-backed`
   generation rule.
3. Later:
   serendipity, analogical planning, mutation, relaxed planning, and
   richer planning-tree reuse.
4. Still unproven at project level:
   cross-session accumulation and a full staged run that someone
   outside the build process actually responds to.
